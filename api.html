API Documentation¶

benchopt¶

Command-line interface to benchOpt

benchopt [OPTIONS] COMMAND [ARGS]...

check-install¶

Check that solvers from benchmark are correctly installed.

benchopt check-install [OPTIONS] BENCHMARK
                       [CLASS_NAMES]...

BENCHMARK¶

Required argument

CLASS_NAMES¶

Optional argument(s)

run¶

Run a benchmark with benchopt.

benchopt run [OPTIONS] BENCHMARK

Options

-l, --local¶

If this flag is set, run the benchmark with the local interpreter.

-r, --recreate¶

If this flag is set, start with a fresh conda env.

-n, --n-rep <n_rep>¶

Number of repetition used to estimate the runtime.

-s, --solver <solver_names>¶

Include <solver_name> in the benchmark. By default, all solvers are included. When -s is used, only listed estimators are included.

-f, --force-solver <forced_solvers>¶

Force the re-installation and run for <solver_name>. This avoids caching effect when adding an estimator.

-d, --dataset <dataset_names>¶

Run the benchmark on <dataset_name>. By default, all datasets are included. When -d is used, only listed datasets are included. Note that <dataset_name> can be a regexp.

--max-samples <max_samples>¶

Maximal number of iteration for each solver [default: 100]

--timeout <timeout>¶

Timeout a solver when run for more than <timeout> seconds [default: 100]
