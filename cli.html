Commande Line Interface (CLI) Documentation¶

benchopt¶

Command-line interface to benchOpt

benchopt [OPTIONS] COMMAND [ARGS]...

Options

-v, --version¶

Print version

check-install¶

Check that a given solver or dataset is correctly installed.

The class to be checked is specified with the absolute path of the file in which it is defined MODULE_FILENAME and the name of the base class BASE_CLASS_NAME.

benchopt check-install [OPTIONS] MODULE_FILENAME
                       BASE_CLASS_NAME

Arguments

MODULE_FILENAME¶

Required argument

BASE_CLASS_NAME¶

Required argument

plot¶

Plot the result from a previously run benchmark.

benchopt plot [OPTIONS] BENCHMARK

Options

-f, --filename <filename>¶

Specify the file to select in the benchmark. If it is not specified, take the latest on in the benchmark output folder.

-k, --kind <kinds>¶

Timeout a solver when run for more than <timeout> seconds

--display, --no-display¶

Whether or not to display the plot on the screen.

--plotly¶

If this flag is set, generate figure as HTML with plotly. This option does not work with all plot kinds and requires to have installed plotly.

Arguments

BENCHMARK¶

Required argument

publish¶

Publish the result from a previously run benchmark.

benchopt publish [OPTIONS] BENCHMARK

Options

-t, --token <token>¶

Github token to access the result repo.

-f, --filename <filename>¶

Specify the file to publish in the benchmark. If it is not specified, take the latest on in the benchmark output folder.

Arguments

BENCHMARK¶

Required argument

run¶

Run a benchmark with benchopt.

benchopt run [OPTIONS] BENCHMARK

Options

--recreate¶

If this flag is set, start with a fresh conda env.

-p, --objective-filter <objective_filter>¶

Filter the objective based on its parametrized name. This can be used to only include one set of parameters.

-s, --solver <solver_name>¶

Include <solver_name> in the benchmark. By default, all solvers are included. When -s is used, only listed estimators are included.

-f, --force-solver <solver_name>¶

Force the re-installation and run for <solver_name>. This avoids caching effect when adding an estimator.

-d, --dataset <dataset_name>¶

Run the benchmark on <dataset_name>. By default, all datasets are included. When -d is used, only listed datasets are included. Note that <dataset_name> can be a regexp.

-n, --max-runs <int>¶

Maximal number of run for each solver. This corresponds to the number of points in the time/accuracy curve.

Default

100

-r, --n-repetitions <int>¶

Number of repetition that are averaged to estimate the runtime.

--timeout <int>¶

Timeout a solver when run for more than <timeout> seconds

Default

100

--plot, --no-plot¶

If this flag is set, do not plot the results.

--pdb¶

Launch a debugger if there is an error. This will launch ipdb if it is installed and default to pdb otherwise.

-l, --local¶

Run the benchmark in the local env. Must have all solvers and dataset dependencies installed.

-e, --env¶

Run the benchmark in a conda env for the benchmark. The env is named benchopt_<BENCHMARK> and all solver dependencies are installed in it.

--env-name <env_name>¶

Run the benchmark in a conda env for the benchmark. The env is named <env_name> and all solver dependencies are installed in it.

Arguments

BENCHMARK¶

Required argument

test¶

benchopt test [OPTIONS] BENCHMARK_DIR

Options

--env-name <NAME>¶

Environment to run the test in. If it is not provided a temporary one is created for the test.
